# CANONICAL TERMINOLOGY

All future UX copy, docs, and AI prompts must use **canonical** terms.  
Forbidden terms are **not** allowed in active product docs or primary routes.

| Canonical term | Forbidden term | Reason |
|----------------|----------------|--------|
| 社交訊號 | emotional analysis | Product is signal intelligence, not emotion typing |
| 社交電量 | compatibility score | Energy model ≠ dating match % |
| 互動耗損 | trauma healing | Not a therapy product |
| 互動節奏 | personality match | Rhythm ≠ MBTI-style pairing |
| 危險互動 | toxic human detector | Pattern risk, not moral judgment on people |
| 北極狐觀察 | therapy guidance | Fox is observer, not counselor |
| 互動壓力 | anxiety diagnosis | Pressure framing, not clinical label |
| 目標觀察 | stalking / surveillance | User-initiated target profile only |
| 訊號檔案 | personality profile | Sliders describe interaction sensitivity |
| 配對牆 | dating wall | Social energy compatibility surface |
| 來電節奏 | love language | De-dating primary narrative |
| 高耗損互動 | red flag person | Describe **patterns**, not identity |

**English guardrails (docs/code comments):**

| Use | Avoid |
|-----|--------|
| social signal intelligence | therapy app, healing AI |
| interaction pressure | emotional healing |
| social energy | compatibility score (as primary label) |
| match credibility | dating score system |
| dangerous interaction pattern | personality analyzer |
