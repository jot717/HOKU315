# TARGET SIGNAL — CONSTITUTION (v1)

## PURPOSE

The system analyzes **interaction compatibility pressure** between:

- **USER SIGNALS** (profile, mine vector, memory, bound-flow context)
- **TARGET SIGNALS** (user-described observation entity: rhythms, demands, patterns)

The **target** is **not** a real diagnosis, moral label, or clinical assessment.

The target is a **signal observation entity**: structured notes the user entrusts to the fox for **guardian-style overlap analysis** against rule-based archetypes.

## IMPORTANT

The system analyzes **interaction risks** and **pressure shapes**.

It does **not** analyze human worth, "bad person" identity, or dating compatibility scores.

## SAFE MODE

- Local JSON only (`runtime_state/target_profile.json`)
- No SNS APIs, no multi-user graph, no embeddings, no Supabase changes for targets

## ALIGNMENT

- Primary loop: **Profile → Target setup → Insight → Guardian output → Memory**
- See [`TARGET_ANALYSIS_FLOW.md`](TARGET_ANALYSIS_FLOW.md), [`TARGET_PROFILE_SCHEMA.md`](TARGET_PROFILE_SCHEMA.md)
