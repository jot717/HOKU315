# PHASE1 UX RESTRUCTURE — UAT NOTES

## Previous confusion problems

Users could click through the UI but often could not answer:

* what the product **is** (guardian world vs generic AI)
* what the fox **does** (protect/filter vs “analyze you”)
* why results **matter** (safety/drain vs abstract insight)
* what to do **next** (bounded actions vs open-ended dashboard)
* why this differs from **AI analysis apps** (judged/scored vs protected/guided)

## What users failed to understand

* Profile as a **signal file** for protection (not “profile for matching”)
* Observation as **guardian scanning** (not “AI reading your personality”)
* Outputs as **warnings + pacing** (not “scores / compatibility”)

## Why old UX failed

* Too many stacked cards resembled an **AI report**
* Analytical sections appeared before a clear **guardian conclusion**
* Too much text and duplicate framing (scan + guard + notes + hero)
* Score-like visuals appeared early in the hierarchy

## New UX hierarchy (PHASE1)

1. Fox presence + immediate emotional guidance  
2. One clear **main warning** headline  
3. **Why** (max 3 bullets)  
4. **Fox guidance** (short action lines)  
5. **Next actions** (large, obvious buttons)

## Emotional target

* protected, lighter, accompanied — **not** evaluated

## UAT pass / fail criteria

### PASS

* User can explain the product as a **guardian / signal safety** system in one sentence
* User understands **why** they entered profile data
* User understands **what observation does** before running it
* After a result, user can name **one risk** and **one next step**

### FAIL

* User describes it as dating / matching / personality scoring
* User cannot explain what to do next
* User feels “judged” or “tested” by the primary screen
