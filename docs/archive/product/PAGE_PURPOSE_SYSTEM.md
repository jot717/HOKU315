# Page purpose system (PHASE1)

Each route has **one** primary job. Copy and navigation must not imply a second product.

| Route | Primary purpose | User should understand |
|-------|-----------------|------------------------|
| `/` | Explain the system in seconds; show the five steps. | “This analyzes social signals and risky interaction patterns.” |
| `/profile` | Build **my** baseline signal profile. | “This is my inputs for how I handle stress and social rhythm.” |
| `/quiz` | Deepen **my** sensitivity model (20 dimensions). | “This is not another profile; it refines how analysis applies to me.” |
| `/target` | Capture **who** is being observed. | “I’m describing the other side of the interaction for risk context.” |
| `/insight` | Deliver the **main value**: structured analysis + one short fox note + next steps. | “I get drain risk, patterns to watch, fit / avoid types, what to do next.” |
| `/match` | Show **relatively safer** social options under vector rules. | “High-drain signals are filtered; cards explain why someone appears here.” |

## Deprecated page narratives (do not reintroduce on these routes)

- Standalone “guardian world” or healing journey as the main story.  
- Duplicate onboarding blocks that repeat the same emotional setup.  
- Multiple competing CTAs that skip the locked order without reason.

## Legacy routes

`/story`, `/unlocks`, `/chat` remain in the app for historical or internal reasons but are **not** part of the primary nav in PHASE1.
